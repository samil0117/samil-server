# 타코야키 박스 — 서버

타코야키 박스로 친구들과 세션을 하려면 **모두가 접속할 서버 하나**가 필요합니다. 이 폴더가 그 서버예요.
서버를 켜면 **주소(예: `https://...`)** 가 하나 생기고, 그 주소를 앱의 **\[접속\]** 칸에 넣으면 끝입니다.

> 서버는 **한 명만** 켜면 됩니다(보통 GM). 나머지 친구들은 앱에 그 주소만 넣으면 돼요.

---

## 가장 쉬운 두 가지 길

### 길 A — 내 PC에서 바로 (무료 · 더블클릭 한 번)

이 폴더의 **`타코야키-서버-시작.bat`** 을 더블클릭하세요. 그러면 알아서:

1. 필요한 것(Node.js)이 있는지 확인하고,
2. 서버를 설치·실행하고,
3. **친구들이 접속할 공개 주소(`https://....trycloudflare.com`)** 를 만들어 화면에 띄워줍니다.

화면에 뜬 주소를 복사해 앱의 **\[접속\]** 칸에 넣으면 됩니다.

- ⚠️ Node.js가 없다면 자동으로 [nodejs.org](https://nodejs.org) 를 열어줘요. **LTS** 버전을 설치(다음다음)한 뒤 `.bat` 을 다시 더블클릭하세요.
- ⚠️ **그 창을 켜 두는 동안에만** 친구들이 접속할 수 있어요(내 PC가 서버라서). 창을 닫거나 PC를 끄면 서버도 꺼집니다.
- ⚠️ 무료 방식이라 **실행할 때마다 주소가 바뀝니다.** 매번 새 주소를 친구들에게 알려주세요. 늘 같은 주소로 24시간 돌리고 싶다면 **길 B**를 쓰세요.

### 길 B — 클라우드에서 24시간 (Railway · 월 약 $5)

PC를 꺼도 서버가 살아 있고, 주소도 **고정**됩니다. 아래 버튼으로 시작하세요.

[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new)

1. 이 폴더를 **내 깃허브(GitHub)** 에 올린 뒤(아래 「깃허브에 올리기」 참고), Railway에서 **New Project → Deploy from GitHub repo** 로 그 저장소를 고릅니다.
2. ⚠️ **꼭** 서비스에 **볼륨(Volume)** 을 추가하고 **Mount path** 를 정확히 **`/app/data`** 로 지정하세요. (이걸 안 하면 재시작 때 계정·세션이 다 사라집니다.)
3. **Settings → Networking → Generate Domain** 으로 주소를 만들어, `주소/health` 가 `{"ok":true}` 면 성공.

더 자세한 그림 설명은 함께 받은 **「서버 호스팅 가이드」** 문서를 보세요(무료 Oracle·Render·내 PC 방법도 있습니다).

---

## 깃허브에 올리기 (길 B에서 씀)

1. [github.com](https://github.com) 가입 → 오른쪽 위 **+ → New repository** → 이름 입력 → **Public** → **Create**.
2. **uploading an existing file** 링크 → 이 폴더 **안의 파일들**(`Dockerfile`, `package.json`, `src` 등)을 끌어다 놓습니다.
   - ⚠️ `node_modules`, `cloudflared.exe` 같은 무거운 파일은 빼고 올리세요(없어도 됩니다).
3. **Commit changes**. 이제 이 저장소를 Railway에 연결하면 됩니다.

---

## 알아두기

- 서버 데이터(계정·세션방·캐릭터·업로드 이미지)는 **`data` 폴더**(클라우드에서는 `/app/data`)에 저장됩니다.
- `data` 폴더에는 **비밀번호 등 개인정보**가 들어가니 **절대 깃허브에 올리지 마세요.** (`.gitignore` 가 막아주지만, 웹에서 직접 끌어 올릴 땐 빼고 올리세요.)
- 환경변수(선택): `SESSION_TTL_DAYS`(로그인 유지일, 기본 30), `CORS_ORIGINS`(허용 주소, 기본 전체). 그대로 둬도 됩니다.
